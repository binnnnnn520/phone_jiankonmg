# v0.2.0 Release Notes

Release date: 2026-05-01

## Summary

This release validates the public two-device monitoring flow on real phones.

- Public HTTPS app URL: `https://app-47-86-100-51.sslip.io`
- Public HTTPS signaling URL: `https://signal-47-86-100-51.sslip.io`
- Camera and viewer pairing uses QR code plus PIN
- Live video streams through WebRTC
- Environment audio starts with monitoring and is played live only
- No audio or video recording is saved
- TURN relay support is configured for remote networks

## User-Facing Changes

- Environment audio is requested by default when the camera starts monitoring.
- The camera falls back to video-only if microphone permission or capture fails.
- The viewer can mute or unmute environment audio locally.
- QR links use the public viewer URL for cross-device use.
- Video defaults were tuned for a clearer and lower-latency remote stream:
  - Balanced: 1280x720 capture target, 20-24 fps, 1.4 Mbps cap
  - Sharp: 1280x720 capture target, 24-30 fps, 2.2 Mbps cap
  - Data saver: 640x360 capture target, 15-20 fps, 450 kbps cap
- WebRTC video offers include startup bitrate hints so clarity ramps up faster after connection.

## Operational Notes

- The browser/PWA camera side must remain open and in the foreground.
- Existing monitoring sessions should be restarted after deployment to pick up new media settings.
- If a QR code points to `localhost`, it is not usable from another device; use the public HTTPS app URL for cross-device testing.
- Further video tuning should be based on measured stats, not subjective delay alone.
